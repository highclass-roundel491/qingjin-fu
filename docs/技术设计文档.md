## 1. 系统架构设计

```
                            ↓
```

### 1.2 技术栈选型

#### 1.2.1 前端技术栈
- 框架：Vue 3.4+
- 构建工具：Vite 5+
- 路由：Vue Router 4
- 状态管理：Pinia
- UI组件库：自定义组件（水墨诗意风格）
- 图表库：ECharts 5+ (vue-echarts)
- HTTP客户端：Axios
- CSS预处理器：原生CSS + CSS变量
- 类型检查：TypeScript (vue-tsc)

#### 1.2.2 后端技术栈
- 框架：FastAPI 0.110+
- Python版本：3.11+
- 异步框架：asyncio + uvicorn
- ORM：SQLAlchemy 2.0+
- 数据验证：Pydantic
- 认证：JWT (python-jose)
- 密码加密：passlib + bcrypt
- 数据库迁移：Alembic
- API文档：Swagger UI (FastAPI自带)

#### 1.2.3 数据库
- 关系型数据库：PostgreSQL 15+
- 缓存：Redis 7+
- 连接池：asyncpg

#### 1.2.4 AI服务
- 主平台：阿里云百炼（dashscope）
- 日常/轻量调用：qwen-flash（0.15元/百万token）
- 复杂评分/创作：qwen-plus（0.8元/百万token）
- 接口方式：OpenAI 兼容接口

#### 1.2.5 部署相关
- 容器化：Docker + Docker Compose
- Web服务器：Nginx
- 进程管理：Supervisor / systemd
- 日志：Python logging + 文件轮转

## 2. 数据库设计

### 2.1 ER图概述
```
用户表 (users) ──┬── 1:N ── 作品表 (works)
                 ├── 1:N ── 收藏表 (favorites)
                 ├── 1:N ── 填词挑战 (challenges / challenge_submissions)
                 ├── 1:N ── 作品评论 (work_comments / work_comment_likes)
                 ├── 1:N ── 关注表 (user_follows)
                 ├── 1:N ── 成就表 (user_achievements)
                 ├── 1:N ── 动态流 (activity_feeds)
                 ├── 1:N ── 学习记录 (learning_records)
                 ├── 1:N ── 飞花令 (feihualing_sessions)
                 ├── 1:N ── 接龙房间 (relay_rooms / relay_players / relay_rounds)
                 └── 1:N ── 限时挑战 (timed_challenge_sessions / timed_challenge_answers)

诗词表 (poems) ──┬── 1:N ── 收藏表 (favorites)

                 └── 1:N ── 作品点赞 (work_likes)

```

### 2.2 核心表结构

#### 2.2.1 用户表 (users)
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    phone VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(50),
    avatar_url VARCHAR(255),
    bio TEXT,
    level INTEGER DEFAULT 1,
    exp INTEGER DEFAULT 0,
    points INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    is_admin BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
```

#### 2.2.2 诗词表 (poems)
```sql
CREATE TABLE poems (
    id SERIAL PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    author VARCHAR(50) NOT NULL,
    dynasty VARCHAR(20) NOT NULL,
    content TEXT NOT NULL,
    translation TEXT,
    annotation TEXT,
    background TEXT,
    appreciation TEXT,
    category VARCHAR(50),
    genre VARCHAR(50),
    tags TEXT[],
    view_count INTEGER DEFAULT 0,
    favorite_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_poems_author ON poems(author);
CREATE INDEX idx_poems_dynasty ON poems(dynasty);
CREATE INDEX idx_poems_category ON poems(category);
CREATE INDEX idx_poems_title ON poems(title);
CREATE FULLTEXT INDEX idx_poems_content ON poems(content);
```

#### 2.2.3 填词题目表 (fill_questions)
```sql
CREATE TABLE fill_questions (
    id SERIAL PRIMARY KEY,
    poem_id INTEGER REFERENCES poems(id),

CREATE INDEX idx_fill_questions_poem_id ON fill_questions(poem_id);
```

#### 2.2.4 答题记录表 (answer_records)

## 3. API设计规范

### 3.1 RESTful API规范

#### 3.1.1 URL设计
- 使用名词复数形式
- 使用小写字母和连字符
- 版本号放在URL中：`/api/v1/`

#### 3.1.2 HTTP方法
- GET：查询资源
- POST：创建资源
- PUT：完整更新资源
- PATCH：部分更新资源
- DELETE：删除资源

#### 3.1.3 状态码
- 200：成功
- 201：创建成功
- 204：删除成功
- 400：请求参数错误
- 401：未认证
- 403：无权限
- 404：资源不存在
- 500：服务器错误

#### 3.1.4 响应格式
```json
{
    "code": 200,
    "message": "success",
    "data": {},
    "timestamp": 1234567890
}
```

### 3.2 认证机制

#### 3.2.1 JWT Token
- Access Token：有效期2小时
- Refresh Token：有效期7天
- Token存储在HTTP-only Cookie或Authorization Header

#### 3.2.2 认证流程
```
1. 用户登录 → 返回 Access Token + Refresh Token
2. 请求API → Header携带 Authorization: Bearer {token}
3. Token过期 → 使用 Refresh Token 刷新
4. Refresh Token过期 → 重新登录
```

### 3.3 分页规范
```
GET /api/v1/poems?page=1&page_size=20

Response:
{
    "code": 200,
    "data": {
        "items": [],
        "total": 100,
        "page": 1,
        "page_size": 20,
        "total_pages": 5
    }
}
```

## 7. 缓存策略

### 7.1 内存缓存 (redis_cache.py)
- 诗词列表/搜索接口响应缓存
- 排行榜数据缓存
- 图谱数据缓存
- 统计数据缓存
- 缓存命中直接返回 JSONResponse（跳过 Pydantic 序列化）

### 7.2 LLM结果缓存 (agent/cache.py)
- author_bio / meter_analysis / allusions / deep_appreciation / verse_analysis
- TTL 6-24h，最多500条
- 内存字典存储

### 7.3 缓存过期策略
- 诗词接口缓存：按请求参数组合键
- 排行榜：按查询参数键
- LLM结果：按 poem_id + context_type 键，TTL 6-24h

## 8. 性能优化

### 8.1 数据库优化
- pg_trgm 扩展 + GIN trigram 索引（title/author/content/tags）
- 只查必要列，避免 ORM 全对象加载
- N+1 查询修复（飞花令等模块）
- LIKE 通配符转义防注入
- 连接池配置（asyncpg）
- EXPLAIN 分析慢查询定位瓶颈

### 8.2 前端优化
- 路由懒加载（所有页面组件均使用 dynamic import）
- SVG 图标系统（文件式，避免内联）
- CSS 变量统一主题
- vue-tsc 类型检查零错误

### 8.3 后端优化
- 缓存命中直接返回 JSONResponse（跳过 Pydantic 序列化）
- 工具并行执行 asyncio.gather
- LLM 结果缓存层
- 请求频率限制（读120/min，写30/min）
- X-Response-Time 响应计时头
- 压测结果：64并发100%成功率，RPS 596-689，热缓存 p95 10-20ms
